How it works

Four steps, no configuration required

Point it at a folder. It reads the code, classifies the risk, checks the obligations that actually apply, and ships you the fix.

01

Scan your code

AST-parsed detection of import openai, from langchain, AgentExecutor(), and every major provider/framework — only in files that actually import them.

02

Classify risk

MINIMAL, LIMITED, HIGH, or UNACCEPTABLE — decided by Annex III use-case domain (hiring, credit, biometrics…), not by technical capability.

03

Check compliance

Per-article status — Met, Partial, Unverified, Missing, Not assessed — against the obligations your risk tier actually triggers.

04

Recommend fixes

Every gap maps to a real, copy-paste template and an exact location to drop it in your project.

Applicability

Do I need this?

✓

Yes, if you

  • Use OpenAI, Anthropic, Google, or any AI API
  • Build chatbots or AI assistants
  • Use LangChain, CrewAI, AutoGen, or LangGraph
  • Deploy AI in the EU or serve EU users
  • Want to avoid fines — up to €15M / 3% of turnover for most obligations, €35M / 7% for prohibited practices
✕

No, if you

  • Don't use AI in your project
  • Only use AI for personal, non-business projects
  • Don't operate in, or serve users in, the EU
Classification

Risk tiers, stamped on every report

HIGH comes from the use-case domain (Annex III), not from having an agent with tools — a résumé-screener is high-risk, an autonomous coding assistant on its own is not.

MINIMAL

Basic AI usage, no user interaction. Almost no rules apply.

e.g. an internal batch-summarization script
LIMITED

AI interacts with users directly. Transparency rules apply (Art. 50).

e.g. a customer-support chatbot
HIGH

Used in an Annex III domain — hiring, credit, biometrics, law enforcement. Full Chapter III compliance required.

e.g. a résumé-screening tool
UNACCEPTABLE

Banned AI practices under Art. 5 — cannot be deployed at all.

e.g. untargeted facial-recognition scraping

FAQ

Common questions

Is this legal advice?
No. It's a technical tool that checks your code. Consult a lawyer for legal advice.
Will this slow down my CI/CD?
No. It takes about 5 seconds on most projects.
What if I'm not in the EU?
If you serve EU users, you still need to comply. The EU AI Act applies to anyone providing AI to EU residents.
What if I find issues?
The tool gives you exact code fixes. Copy the templates into your project and re-run the scan.
Can I use this in production?
Yes. Add it to your CI/CD pipeline to catch issues automatically on every PR.
command not found: compliance-agent — now what?

Command reference

Every command, grouped by what you're trying to do

Run compliance-agent scan --help any time for the full, exhaustive flag list.

shell
# the core command — . means current folder compliance-agent scan . # output formats compliance-agent scan . --format markdown # default, terminal-readable compliance-agent scan . --format json # for CI / tooling, to stdout compliance-agent scan . --format sarif --output results.sarif # GitHub code scanning compliance-agent scan . --format pdf --output report.pdf compliance-agent scan . --format html --output dash.html # interactive file # narrow what gets scanned compliance-agent scan . --severity high compliance-agent scan . --exclude "tests/*" --exclude "docs/*" compliance-agent scan . --include "src/*" # output shape compliance-agent scan . --quiet # summary only compliance-agent scan . --no-color compliance-agent scan . --verbose compliance-agent scan . --fix # show how to fix each problem inline

Exit codes: 0 success · 1 --fail-on threshold met · 2 usage error.

shell
compliance-agent recommend . --output ./fixes # copies real fix templates in compliance-agent recommend . --format json # machine-readable

Every recommendation names the rule, shows the template, and says exactly where to put it.

shell
compliance-agent report . --output audit-2026.pdf compliance-agent report . --format html --output audit-2026.html

A shareable file for a lead, a client, or an audit trail — same data as scan, formatted to hand off.

shell
compliance-agent scan . --format json -o before.json # ...make changes... compliance-agent scan . --format json -o after.json compliance-agent diff before.json after.json compliance-agent diff before.json after.json --fail-on-regression # CI gate

See whether a change made compliance better or worse — and gate a PR on it.

shell
pip install "compliance-agent[web]" compliance-agent serve . # opens a local dashboard, same "Official Journal" styling

Requires the web extra. Same report, browsable — findings table, coverage ledger, scan history and trend.

yaml
# GitHub Action — one step scans, gates the build, writes SARIF permissions: security-events: write contents: read steps: - uses: actions/checkout@v4 - uses: latreon/compliance-agent@v0 with: path: . fail-on: high - uses: github/codeql-action/upload-sarif@v3 if: always() with: sarif_file: ${{ steps.compliance.outputs.report }} # or, plain shell step: compliance-agent scan . --ci --fail-on high

Findings and gaps appear directly in the repo's Security tab. Full reference, non-GitHub CI (GitLab, CircleCI), pre-commit hook: the CI/CD doc.

shell
compliance-agent upgrade # updates in place: detects uv / pipx / pip compliance-agent upgrade 0.1.2 # pin a specific version compliance-agent version compliance-agent --version # -V: quick check, then exit

Declare scan defaults once in a compliance.yaml — CLI flags always win, a broken config is a hard error. Full schema: the Configuration doc.
